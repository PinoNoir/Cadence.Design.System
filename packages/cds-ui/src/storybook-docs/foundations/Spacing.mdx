# CDS Spacing

> All spacing values use `rem` units (root em).

> `rem` unit is only relative to the HTML document's root element.

> `em` unit is only relative to the immediate parent of the targeted element.

## Spacing Tokens

Read more about using CSS custom properties [here <Icon icon="mdi:open-in-new"/>](https://developer.mozilla.org/en-US/docs/Web/CSS/Using_CSS_custom_properties" 'title').

```css
--cds-space-48: 2.75rem /* 48px */ --cds-space-40: 2.5rem /* 40px */ --cds-space-32: 2rem /* 32px */ --cds-space-24:
  1.5rem /* 24px */ --cds-space-16: 1rem /* 16px - Base size */ --cds-space-12: 0.75rem /* 12px */ --cds-space-8: 0.5rem
  /* 8px */ --cds-space-4: 0.25rem /* 4px */ --cds-space-2: 0.125rem /* 2px */ --cds-space-none: 0
  /* 0px - Can be used as a reset */;
```

## Using CSS Utility Classes

If using the `@stretto/cds-scss` package, you can use the following utility classes in your markup (similar to Bootstrap utilities):

<Markdown>
  {`
| Utility Class | CSS Property    | Value  |
|---------------|-----------------|--------|
| mt-[n]        | margin-top:     | [n]rem |
| mb-[n]        | margin-bottom:  | [n]rem |
| mx-[n]        | margin-block:   | [n]rem |
| my-[n]        | margin-inline:  | [n]rem |
| pt-[n]        | padding-top:    | [n]rem |
| pb-[n]        | padding-bottom: | [n]rem |
| px-[n]        | padding-block:  | [n]rem |
| py-[n]        | padding-inline: | [n]rem |
`}
</Markdown>
`n = none, 2, 4, 8, 12, 16, 24, 32, 40, 48`

`x = horizontal spacing (left and right)`

`y = vertical spacing (top and bottom)`
